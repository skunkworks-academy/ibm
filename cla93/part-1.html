Db2 Product & Platform Overview

  • Supported OS: AIX, Linux, Windows, IBM i, z/OS
  • Editions: Community, Starter, Standard, Advanced
  • Understand licensing limits (cores, memory, features)
  • VRMF versioning model (Version.Release.Mod.Fix)
  • On-prem, Cloud, and Containerized deployment models

Db2 Tools & Connectivity

  • Command Line Processor (CLP) – SQL and Db2 commands
  • CLPPlus – scripting and formatted output
  • Data Management Console (DMC)
  • IBM Data Server Drivers (JDBC, ODBC, CLI)
  • Db2 Connect for z/OS and IBM i connectivity

Db2 Instance Architecture

  • Instance = Database Manager environment
  • Multiple databases per instance
  • DBM CFG vs DB CFG
  • EDUs (Engine Dispatchable Units)
  • Environment & registry variables (db2set)

Core DBA Operational Tasks

  • Create / Drop instance (db2icrt, db2idrop)
  • Start / Stop instance (db2start, db2stop)
  • Update DBM configuration
  • Quiesce and unquiesce instance
  • Validate prerequisites (db2prereqcheck)

Key Takeaways

  • Understand instance-level vs database-level configuration
  • Choose the correct edition for workload requirements
  • Use proper client connectivity for enterprise integration
  • Maintain separation between environment, instance, and database scope
  • Foundation knowledge determines production stability
