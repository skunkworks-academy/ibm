How to use this page

Answer first without peeking. Then enable Exam Reinforcement Mode to reveal “why correct / why wrong”.

These questions are designed to catch the exact misunderstandings that cause broken installs and split-brain nightmares.
1) What is the primary purpose of quorum in Storage Scale?

A) Improve read performance by choosing the fastest nodes
B) Prevent split-brain by ensuring only one partition continues I/O
C) Reduce network traffic by limiting management messages
D) Guarantee the cluster stays online even if most nodes fail

Correct: B
Why correct: Quorum is an integrity safeguard: it prevents two partitions from writing independently.
Why wrong: A/C are performance/optimization statements; D confuses integrity with availability (quorum may stop the cluster to prevent corruption).
2) Which statement best describes an NSD (Network Shared Disk)?

A) A NAS share exported via NFS
B) A raw disk that can only be used by one node at a time
C) A Storage Scale abstraction that allows disks to be shared and accessed across nodes
D) A file system mounted over SMB

Correct: C
Why correct: NSD is a protocol/abstraction used by Storage Scale for shared block access, not “just a disk”.
Why wrong: A/D are file-level sharing protocols; B contradicts the shared-disk model and NSD server/client design.
3) Why must kernel-devel and kernel-headers match the running kernel on Linux?

A) To enable NFS exports
B) To build the Storage Scale portability layer for that kernel version
C) To prevent users from deleting files
D) To automatically configure NSDs

Correct: B
Why correct: The portability layer (kernel integration) must be built against the running kernel interfaces.
Why wrong: A/D are unrelated; C is a permissions/ACL topic.
4) Where are Storage Scale administrative commands typically located on Linux?

A) /usr/bin
B) /opt/gpfs/bin
C) /usr/lpp/mmfs/bin
D) /var/mmfs/gen

Correct: C
Why correct: Storage Scale command binaries/scripts are under /usr/lpp/mmfs/bin.
Why wrong: A is generic; D contains generated/config state (not binaries); B is a common but not the intended standard path here.
5) A cluster network partition happens. What is the “safe” outcome Storage Scale aims for?

A) Both sides keep writing and reconcile later
B) The side that retains quorum continues I/O; the other side stops I/O
C) The node with fastest disks becomes manager automatically and forces others to follow
D) The cluster always stays up because disk leasing prevents problems

Correct: B
Why correct: Only one partition should remain active for writing; otherwise split-brain corruption occurs.
Why wrong: A is the classic corruption recipe; C is fantasy; D overstates disk leasing (it helps, but quorum remains a core integrity mechanism).
6) In the Linux install flow, what does the self-extracting installer primarily do?

A) Automatically creates your cluster and file systems
B) Stages installable packages under /usr/lpp/mmfs/<version>
C) Formats disks into NSDs immediately
D) Enables SMB and NFS exports by default

Correct: B
Why correct: It extracts/stages packages; you then install RPMs and perform cluster creation separately.
Why wrong: A/C/D are later steps or optional features, not what extraction primarily does.
7) A learner edits /var/mmfs/gen/mmsdrfs directly to “fix config quickly.” What’s the best answer?

A) Good practice; it’s a standard config file
B) Fine if you restart the daemon afterwards
C) Not supported; use Storage Scale commands because the file is replicated/generated cluster-wide
D) Only allowed on non-quorum nodes

Correct: C
Why correct: Cluster-wide state is managed via commands; direct edits risk inconsistency and are explicitly discouraged.
Why wrong: A/B/D normalize an unsupported shortcut and ignore replication/consistency risks.
8) What is “pagepool” in Storage Scale?

A) The OS file cache used by applications
B) Storage Scale’s own cache for data/metadata buffers and communication buffers
C) A swap partition reserved for GPFS
D) A set of disks dedicated to descriptor quorum

Correct: B
Why correct: Storage Scale uses its own caching mechanisms (pagepool) rather than relying on the OS file cache.
Why wrong: A confuses caches; C/D are different subsystems.
9) Which command is most directly used to see daemon state across nodes?

A) mmlsattr
B) mmgetstate -aLs
C) mmcrnsd
D) mmchdisk

Correct: B
Why correct: mmgetstate -aLs shows Storage Scale daemon state for nodes (and highlights quorum nodes).
Why wrong: A relates to file attributes; C creates NSDs; D changes disk attributes inside a filesystem context.
10) You add NSD capacity to a capacity-licensed edition. What changes?

A) Nothing; licensing is per client node only
B) Your licensing footprint changes because capacity licensing is based on total NSD capacity in the cluster
C) Only the GUI license changes, not storage
D) The cluster must be rebuilt

Correct: B
Why correct: In capacity-based licensing, the cluster’s total NSD capacity determines the licensing footprint.
Why wrong: A is false (clients may not require license, but capacity does); C is irrelevant; D is unnecessary.
