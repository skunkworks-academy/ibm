What Storage Scale is (and isn’t)

Storage Scale (formerly GPFS) is a clustered file system designed for concurrent access from multiple nodes, scaling capacity and performance while keeping a single namespace.

Key mental model: Storage Scale is not a classic “single filer”. There is no single metadata server bottleneck, and the system relies on quorum to prevent split-brain.
Core components you must name without hesitation
  • Nodes (clients, NSD servers, quorum nodes)
  • NSDs (Network Shared Disks) exporting storage to the cluster
  • Daemon + kernel extension (I/O path + VFS integration)
  • Pagepool (Storage Scale cache, not OS page cache)
  • File system descriptor (replicated metadata on disks)
Exam trap: “Quorum improves availability” (Nope.)

Quorum primarily protects integrity by preventing multiple partitions from writing independently. The system would rather stop than corrupt.
