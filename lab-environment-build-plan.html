Practical Lab Environment – Build Plan

This document defines the hardware, software, setup effort, timelines, and pricing models for delivering hands-on IBM DataPower and IBM App Connect Enterprise labs in instructor-led cohorts. It is designed for repeatable, production-grade training delivery.

Fundamentals of IBM App Connect Enterprise V11

Integration

Baseline Infrastructure (Standardised)

  • Compute: 16–24 vCPU per 8–10 students
  • Memory: 64–96 GB RAM
  • Storage: 500 GB SSD (snapshots + fast I/O)
  • Hypervisor: VMware ESXi, Proxmox, KVM, or Cloud VM
  • Networking: Private lab network + NAT + fixed student IPs
  • Access: Instructor jump host, isolated student domains

Course-Specific Lab Stacks

WE751 – DataPower Development

  • IBM DataPower Gateway V7.5.1.x (2 vCPU / 4–6 GB RAM)
  • Ubuntu 14.04 LTS Student VM
  • Tools: Firefox, SoapUI, cURL
  • Preloaded Booking & Baggage mock services

WE753 – AAA, OAuth, OIDC

  • All WE751 components
  • LDAP backend (OpenLDAP)
  • OAuth Web Token Service
  • Node.js OAuth/OIDC client
  • Per-student SSL certificates & secrets

WM669 – App Connect Enterprise

  • Windows VM (4 vCPU / 8 GB RAM)
  • IBM App Connect Enterprise Toolkit V11
  • Standalone Integration Server
  • Lab files in C:\labfiles

Operational Recommendation

For Feb–Mar 2026 cohorts, the optimal model is time-boxed lab rental. It minimises risk, preserves intellectual property, enables rapid resets, and maximises margin while maintaining delivery quality.
